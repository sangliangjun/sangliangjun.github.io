2025-03-05｜Unknown｜CVPR2025｜⭐️⭐️
http ://arxiv.org/abs/2503.03803v1https ://egolife-ai.github.io/.
EgoLife项目旨在开发一个以自我为中心的生活助手，利用AI技术提升个人效率。该项目通过一项全面的数据收集研究，六名参与者在一个星期内共同生活，记录他们的日常活动，包括社交、购物、烹饪等。每位参与者佩戴MetaAria眼镜，持续捕捉约8小时的自我视角视频，同时使用15个摄像头和2个毫米波设备，收集同步的第三人称视角数据。这些数据的丰富注释为构建先进的多模态自我中心AI助手奠定了基础，并引入了新任务，以推动长期的自我中心生活辅助技术的发展。
EgoLife项目的核心方法包括三个主要技术挑战的解决方案。首先，开发稳健的视觉-音频模型，以处理自我中心的数据。其次，实现身份识别，以便在多个人的环境中准确跟踪参与者。最后，支持长时间上下文的问题回答，处理复杂的时间序列数据。为此，EgoButler系统被提出，其中包括EgoGPT和EgoRAG两个组成部分。EgoGPT是一个经过训练的全模态模型，专注于自我中心数据的理解，而EgoRAG则是一个基于检索的组件，支持超长上下文问题的回答。通过这些技术，EgoLife项目能够有效地分析和理解参与者的日常生活，并提供个性化的建议和反馈。
实验部分主要通过收集和分析EgoLife数据集来验证所提出方法的有效性。数据集包含300小时的多模态自我中心视频及其丰富的注释，涵盖了参与者在一周内的各种活动。研究者们设计了一系列问题，评估EgoButler系统在处理长上下文问题时的表现。通过与现有模型的比较，实验结果展示了EgoButler在准确性和上下文理解方面的优势。此外，实验还揭示了在多模态数据整合和个性化响应生成中的关键因素和瓶颈，为未来的改进提供了指导。
在EgoLife项目中，研究团队希望通过一种新型的AI助手来帮助人们更好地管理日常生活。为了实现这一目标，他们让六个人在一个星期内一起生活，记录下他们的每一天，包括吃饭、购物和社交活动。每个人都佩戴了一种特殊的智能眼镜，这些眼镜可以捕捉到他们的视角和周围环境的声音。研究团队还使用多个摄像头来从不同的角度记录这些活动。通过分析这些数据，他们开发了一个称为EgoButler的系统，这个系统可以根据过去的活动和对话，回答人们的问题，比如“我上次在哪里放过剪刀？”或者“我今天喝了多少水？”这样，EgoLife项目希望未来的AI助手能够更好地理解和帮助人们的日常生活。
2025-03-06｜UTokyo,FDU,ZJU,NJU,ShanghaiAILab｜⭐️🟡
http ://arxiv.org/abs/2503.04250v1https ://github.com/OpenGVLab/vinci
本研究提出了一种新型的可穿戴实时智能助手——Vinci，该系统基于一种名为EgoVideo-VL的视觉语言模型，旨在为用户提供全面的AI支持。Vinci的设计理念是实现实时的上下文感知，支持多种便携设备，包括智能手机和可穿戴相机。系统的核心功能包括场景理解、时间定位、视频摘要以及未来规划。为了增强实用性，Vinci还集成了一个内存模块，能够在处理长视频流时保留上下文历史信息。此外，Vinci具备生成模块和检索模块，前者用于生成视觉行动演示，后者则能够提供相关的如何操作视频，帮助用户掌握新技能。与现有依赖专用硬件的系统不同，Vinci具有硬件无关性，能够在多种设备上部署。
Vinci的实现依赖于EgoVideo-VL模型，该模型结合了以用户视角为中心的视觉基础模型和大型语言模型（LLM），以实现强大的多模态理解和交互。具体方法包括：
输入处理模块：接收实时视频和音频流，并进行语音转文本，将音频转换为可供后续处理的文本格式。
EgoVideo-VL模型：该模型处理视觉和文本输入，生成响应。它专注于用户的第一人称视角，能够理解用户的意图和环境。
内存模块：存储和检索历史上下文，以支持时间推理和个性化互动，确保系统能够理解用户的长期活动。
生成模块：生成视觉指导视频，帮助用户更好地理解和执行任务。
检索模块：从数据库中检索相关的教学视频，为用户提供额外的学习资源。
为了评估Vinci的性能，研究团队进行了多项实验，包括定量评估和用户研究。首先，使用多个公共基准测试来验证EgoVideo-VL模型在视觉语言推理和上下文理解方面的优越性。接着，开展了一系列用户研究，分析Vinci在实际应用中的有效性和适应性。参与者在多种场景中与Vinci进行互动，收集反馈以评估其功能性。结果显示，90%的用户对Vinci的功能表示满意，85%的人认为使用该系统能够提升生活质量和工作效率。这些实验结果表明，Vinci在便携式、实时的以用户为中心的AI系统中具有重要的潜力。
Vinci的工作原理可以简单理解为一个智能助手，能够实时理解和回应用户的需求。它通过一个“输入处理模块”来接收用户的声音和视频，然后将这些信息传递给核心的EgoVideo-VL模型。这个模型会分析周围的环境，并结合用户的语音指令，生成合适的响应。为了帮助用户记住过去的事情，Vinci还有一个“内存模块”，可以记录用户的活动历史，并在需要时提供相关信息。此外，Vinci还可以生成视频演示，告诉用户如何完成某个任务，或者从互联网检索相关的教学视频。这样，无论用户在做什么，Vinci都能提供及时的帮助，让生活和工作变得更轻松有效。
2025-03-06｜ECNU,MeituanInc.,FDU,PazhouLab
http ://arxiv.org/abs/2503.04167v1https ://github.com/Yufang-Liu/visual_modality_role
本研究探讨了视觉模态在多模态数学推理中的角色，尤其是其在现有模型中的应用及效果。尽管多模态数学推理的研究逐渐增多，视觉信息的作用却未被充分挖掘。我们的发现表明，当前的多模态数学模型对视觉信息的利用极其有限，模型的性能在视觉信息缺失或改变时几乎没有显著变化。这一现象主要归因于文本信息的主导地位，模型往往依赖文本中的提示而非视觉信息。为此，我们提出了HC-M3D数据集，旨在通过要求模型依赖图像进行问题求解，来评估其视觉感知能力的局限性。此外，我们观察到，现有模型在处理视觉信息时的敏感性不足，这为未来的研究提供了新的方向。
本研究采用了以下几种方法来评估视觉模态在数学推理中的重要性：
数据集构建：我们创建了HC-M3D数据集，包含1,851个样本，确保每个问题都依赖于相应的图像，且针对429个问题设计了相似但有所改变的图像，以测试模型对视觉差异的敏感性。
模型训练与评估：在训练过程中，我们对主流数学模型进行了标准化，采用三阶段训练方法，确保模型能有效学习图像与文本的对应关系。
视觉模态扰动实验：通过打乱图像与文本的对应关系，观察模型在不同情况下的表现，评估其对视觉信息的依赖程度。
实验部分对多种主流模型在HC-M3D数据集上的表现进行了评估。我们首先对模型在标准图像、随机图像和无图像情况下的表现进行了比较，发现大多数模型在图像被打乱或移除时，性能变化不大，显示出它们对视觉信息的依赖性较弱。此外，尽管存在多种视觉编码器的组合，但在数学推理任务中，模型的性能提升有限，表明在这一领域，视觉信息的作用被高估了。我们还通过对现有数据集的分析，发现许多问题可以仅通过文本信息得出答案，进一步验证了模型在视觉推理上的不足。
在这项研究中，我们主要想了解视觉信息在数学推理中的重要性。我们创建了一个新的数据集，里面的每个问题都需要依赖图像来找出答案。我们通过改变图像来测试模型的反应，看看它们是否能识别出这些变化。我们还对多种数学模型进行了训练和测试，结果发现，即使把图像换成其他的，很多模型的表现也几乎没有变化，这意味着它们并不真正依赖视觉信息。我们还发现，很多问题其实仅靠文本就能回答，这让我们意识到现有模型在理解图像方面的能力远不如我们想象的那么强。因此，这项研究为提升模型对视觉信息的依赖性提供了新的思路和挑战。
2025-03-05｜GoogleDeepMind,UCBerkeley
http ://arxiv.org/abs/2503.03840v1
本研究旨在评估当前最先进的视觉语言模型（VLMs）在几何理解方面的能力，特别是它们是否能够独立于其他认知能力（如推理和世界知识）来理解基本的几何概念。研究借鉴了认知科学中的方法，通过对比美国成年人和亚马逊土著群体的表现，探讨了不同文化和教育背景对几何理解的影响。结果表明，VLMs在几何任务上的表现普遍低于这两组人类参与者，尽管在某些几何概念上表现较好。此外，VLMs在需要进行心理旋转的任务中表现出更大的脆弱性，提示其几何理解的稳定性远不及人类。这项研究揭示了人类和机器在几何理解的来源上存在显著差异，强调了教育和物理世界互动对几何理解的重要性。
本研究采用了多种方法来评估VLMs和人类在几何理解方面的能力。首先，研究团队从Dehaene等（2006）和Franco&Sperry（1977）的工作中复制了几何刺激，创建了包括点、线、平行、图形、全等和对称等六个概念的黑白图像。每个刺激包含六幅图像，其中一幅图像违反了核心几何概念。其次，针对VLMs的实验，使用了GeminiPro1.5模型，给定的提示要求模型选择“不同的那一幅图像”，并提供解释。每组试验随机进行了20次，以确保结果的可靠性。对于人类参与者，研究招募了15名美国成年人，要求他们完成与VLMs相似的任务，选择“不同的那一幅图像”。参与者在完成任务前接受了两次训练，并在无反馈的情况下完成了53个随机试题。
实验部分包括对VLMs和人类在几何理解任务中的表现进行对比。首先，VLMs的表现被评估在多次试验中，结果显示其在大多数几何刺激上的表现低于人类参与者。特别是在需要进行心理旋转的任务中，VLMs的表现明显较差，而人类参与者在相同条件下的表现相对稳定。此外，研究还考察了不同角度的刺激对模型表现的影响，发现VLMs在需要进行心理旋转的情况下表现最差。相反，尽管人类参与者在不同旋转角度下的表现有所波动，但总体上并未受到显著影响。这些结果表明，VLMs在几何理解方面存在明显的局限性，特别是在与人类的实际经验和教育背景相比时。
在这项研究中，科学家们想知道现代人工智能（VLMs）是否能够理解简单的几何图形，比如点、线和形状。他们使用了一种特别的实验方法，先是让AI和人类看一组图像，其中有一幅图像与其他图像不同。然后，他们要求AI和人类选择出不同的那幅图像，并解释理由。为了确保实验的公平性，科学家们进行了多次测试，并随机改变图像的顺序。人类参与者在实验前还接受了训练，以帮助他们更好地完成任务。结果显示，尽管AI在某些情况下表现不错，但整体上它的表现不如人类，尤其是在需要旋转图像时，AI的表现更差。这表明人类的几何理解能力不仅来自书本知识，还与他们的生活经验和实际操作密切相关。
2025-03-06｜BaiduInc
http ://arxiv.org/abs/2503.04065v1https ://github.com/PaddlePaddle/PaddleMIX
随着数字化进程的快速发展，各类文档图像在生产和日常生活中得到广泛应用，对文档图像内容的快速准确解析需求日益迫切。为此，本文提出了PP-DocBee，这是一种新颖的多模态大型语言模型，旨在实现端到端的文档图像理解。该模型通过开发针对文档场景的数据合成策略，建立了一个多样化的数据集，以提高模型的泛化能力。同时，采用了一些训练技术，如动态比例采样、数据预处理和OCR后处理策略。经过广泛评估，PP-DocBee在英语文档理解基准上取得了最先进的结果，并在中文文档理解方面超越了现有的开源和商业模型。该模型的源代码和预训练模型已公开，便于后续研究和应用。
PP-DocBee的核心方法包括几个关键步骤：首先，提出了一种数据合成策略，通过构建477k高质量的中文文档理解数据集PPInfinityDocData，来提升模型的训练效果。其次，采用了多种训练技巧，包括：
动态比例采样：对不同数据源和类型分配不同的采样权重，以优化训练过程，确保高质量数据的比例更高。
数据预处理：在图像处理阶段，采用小块图像分割的方法，将图像分解为多个视觉标记，以便于模型理解和处理图像内容。
OCR后处理：在推理阶段，将OCR识别的文本作为辅助信息，提升模型在文本清晰且有限的图像上的性能。通过这些方法，PP-DocBee显著提升了对复杂文档图像的处理和理解能力。
在实验部分，PP-DocBee的训练数据涵盖了多种文档理解数据集，包括一般的视觉问答（VQA）图像、OCR图像、图表、丰富文本文档等。模型在约500万样本的基础上进行了16k次迭代训练，使用单个A800GPU约耗时两天。实验结果显示，PP-DocBee在多个基准测试中表现优异，尤其在文本丰富的图像任务上，取得了81.2的高分。通过与现有的多模态大型语言模型（MLLMs）进行比较，PP-DocBee在多个任务上都展现出卓越的性能，尤其在中文文档理解方面，显著超越了其他模型。实验还进行了消融研究，验证了数据合成策略和动态比例采样的有效性，进一步证明了这些创新方法对模型性能的提升作用。
PP-DocBee的工作原理可以简单理解为几个步骤的结合。首先，它通过一种特别的方法生成大量高质量的中文文档数据，确保模型在学习时能接触到丰富多样的内容。接着，在训练过程中，模型会根据不同类型的数据，合理分配学习的重心，这样可以更好地掌握重要的信息。此外，PP-DocBee在处理图像时，会把图像切分成小块，像拼图一样，让模型更容易理解每个部分的内容。最后，它还利用OCR技术提取图像中的文本信息，以帮助回答问题。当有清晰的文字时，这种方法能够显著提升模型的表现。通过这些策略，PP-DocBee能够更有效地理解和处理各种复杂的文档图像。
2025-03-06｜NVIDIA,RutgersU,UCBerkeley,MIT,NJU,KAIST
http ://arxiv.org/abs/2503.04130v1https ://research.nvidia.com/labs/lpr/storm
近年来，视频基础的多模态大语言模型（Video-LLMs）在视频理解方面取得了显著进展，能够将视频处理为一系列图像帧。然而，现有的方法往往独立处理每一帧，缺乏明确的时间建模，限制了其捕捉动态模式的能力，尤其是在处理长视频时。为了解决这些问题，我们提出了一种新的架构——STORM（SpatiotemporalTOkenReductionforMultimodalLLMs），它在图像编码器与大语言模型（LLM）之间引入了专门的时间编码器。该时间编码器利用Mamba状态空间模型整合时间信息，生成丰富的表示，保留了整个视频序列中的帧间动态。这种增强的编码不仅提升了视频推理能力，还实现了有效的令牌减少策略，显著降低了对LLM的计算需求，而不牺牲关键的时间信息。
STORM的核心是Mamba基础的时间投影器，它通过在图像编码器与LLM之间引入一个时间编码器，有效整合视频帧之间的时间信息。具体方法包括：
时间编码器：使用Mamba状态空间模型，处理输入序列，捕捉时间动态。
令牌压缩策略：引入时间平均池化和空间平均池化，减少令牌数量，同时保留重要信息。
训练时令牌压缩：在训练过程中应用时间和空间池化，减少输入给LLM的令牌数量，优化计算效率。
测试时令牌采样：在测试阶段，进一步减少输入令牌数量，通过选择性采样保持模型性能。
通过这些方法，STORM能够在处理长视频时显著提高效率和准确性，同时减少计算开销。
我们对STORM在多个长视频理解基准上进行了广泛评估，验证其性能和效率。实验结果显示，STORM在MVBench、MLVU和LongVideoBench等基准上均取得了最先进的结果，提升幅度超过5%。相比于传统方法，STORM在处理长视频时计算成本降低了8倍，解码延迟减少了2.4到2.9倍。我们还进行了消融实验，评估不同令牌压缩策略的有效性，结果表明，结合时间池化和测试时采样的STORM模型在保持性能的同时，令牌数量减少至25%。这些实验结果表明，STORM在长视频理解任务中具备卓越的表现和高效的计算能力。
STORM方法可以简单理解为一个智能的视频处理系统，它在分析视频时，不仅关注每一帧的画面，还特别注意帧与帧之间的关系。首先，它使用一种叫做Mamba的技术，将时间信息直接融入每一帧的表示中。接着，STORM采用了两种方法来减少需要处理的数据量：一种是通过平均相邻帧的信息来压缩数据，另一种是在测试时只选择重要的帧进行分析。这样，STORM在处理长视频时，能够更快地得到结果，同时保持较高的准确性。这种方法的优势在于，它能有效减少计算负担，让AI更好地理解和生成视频内容。

在多模态学习领域，文本与音频的结合已成为重要的研究方向。然而，现有方法通常忽视了模型在面对新数据集时的泛化能力，这可能导致灾难性遗忘。为了解决这一问题，本文提出了一项新任务，即文本-音频增量学习（TAIL），旨在实现跨数据集的音频-文本检索。我们提出了一种新的方法，称为PTAT（PromptTuningforAudio-TextIncrementalLearning），该方法通过优化模型参数，结合音频-文本相似性和特征蒸馏模块，有效缓解了灾难性遗忘的问题。通过在多个数据集（如AudioCaps、Clotho、BBCSoundEffects和Audioset）上进行基准测试，我们的方法在性能上显著优于现有的增量学习方法，特别是在对旧数据集的遗忘抵抗能力方面表现突出。
PTAT方法主要包括两个核心模块：音频-文本提示生成（ATPG）和音频-文本相似性与特征蒸馏（ATSFD）。具体方法如下：
音频-文本提示生成（ATPG）：
该模块生成一系列互相关联的音频和文本提示，通过提示调优来高效优化模型参数。
使用全局音频提示和深度文本提示，以增强模型对新任务的适应能力，同时保持旧知识的稳定性。
音频-文本相似性与特征蒸馏（ATSFD）：
在每个增量步骤中，利用之前模型的输出作为新模型的目标，进行知识蒸馏。
通过计算音频和文本之间的相似性，确保新数据集的学习不会干扰到已学知识，从而减轻遗忘现象。
我们在多个数据集上对PTAT进行了实验，评估其在音频-文本检索任务中的表现。实验设置包括四个增量步骤：首先在AudioCaps上训练，然后依次在Clotho、BBCSoundEffects和Audioset上进行训练。我们使用Recall@1、Recall@5和Recall@10等指标来评估模型的检索性能。实验结果显示，PTAT在各个数据集上均表现优异，尤其在对旧数据集的记忆保持能力上，明显优于传统的全参数微调方法。此外，实验还探讨了不同提示层次和特征蒸馏策略对模型性能的影响，进一步验证了PTAT方法的有效性和高效性。
PTAT方法的核心在于如何让模型在学习新数据时，仍然能记住之前学到的知识。它通过两个主要步骤来实现这一点。首先，音频-文本提示生成模块会创建一些“提示”，这些提示帮助模型更好地理解音频和文本之间的关系。想象一下，这就像给模型提供了一些小提示，帮助它在学习新内容时不忘记旧的内容。其次，音频-文本相似性与特征蒸馏模块则确保模型在学习新数据时，依然能参考之前学到的知识。通过这种方式，模型不仅能适应新数据，还能有效避免遗忘，保持较好的性能。
