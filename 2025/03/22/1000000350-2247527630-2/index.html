主编｜庄才林（CailinZhuang）技术支持｜胡耀淇（YaoqiHu）｜编辑支持｜张奇佳（QijiaZhang）发布日期｜2025年3月10日｜周一
2025-03-07｜OffenburgU,HerrenknechtAG,UMannheim｜⭐️🟡
http ://arxiv.org/abs/2503.05333v1http ://www.physics-gen.org
本研究旨在探讨生成模型在图像到图像转换中的潜力，特别是在物理关系的预测方面。尽管在图像修复和风格转移等视觉任务中已经取得了显著进展，但在物理模拟领域的应用仍然相对较少。研究团队构建了一个包含30万个图像对的数据集，涉及三种不同的物理模拟任务，旨在回答两个核心问题：生成模型是否能够从输入输出图像对中学习复杂的物理关系，以及通过替代基于微分方程的模拟，能够实现多大的速度提升。初步评估显示，现有模型在速度上具有潜力，但在物理准确性方面存在显著局限，这突显了对新方法以确保物理正确性的需求。
为了解决上述问题，研究团队提出了PhysicsGen基准，系统评估生成模型在学习物理模拟任务中的能力。该基准包括以下几个关键组成部分：
多样性物理问题：选择了三种不同类型的物理现象，涵盖了波传播、镜头畸变和运动动态预测等领域，以确保模型在多种情况下的适用性。
数值解法策略：结合了迭代和闭式解法，评估模型对不同计算方法的适应能力。这有助于检验生成模型在处理不同复杂度问题时的灵活性。
易于评估的指标：采用简单且有效的评估标准，以便于对模型性能进行直观的比较和分析。通过这些方法，研究团队能够全面评估生成模型在不同物理任务中的表现，并为后续的研究提供有价值的基准和数据支持。
在实验部分，研究团队针对三项物理模拟任务进行了系统的评估。首先，城市声音传播任务模拟了声波在城市环境中的传播行为，生成了10万个基于微分方程的输入输出图像对。其次，镜头畸变任务利用Brown-Conrady模型生成了5万个面部图像，旨在测试生成模型对图像几何变化的捕捉能力。最后，滚动与弹跳运动任务生成了5万个图像对，评估模型对物体运动动态的预测能力。实验结果表明，尽管生成模型在速度上表现出色，但在物理准确性方面仍面临挑战，尤其是在处理高阶物理关系时，模型的表现显著下降。
在这项研究中，研究者们想要看看一些智能算法（生成模型）能否从图像中学习到物理规律，并用这些规律来预测物体的行为。他们首先建立了一个包含30万个图像对的大型数据库，这些图像展示了不同的物理现象，比如声音在城市中的传播或镜头造成的图像扭曲。接着，他们设计了一些方法来测试这些算法的表现。具体来说，他们选了三种不同的物理问题，并用简单的方式来评估算法的表现。最后，他们发现虽然这些算法在速度上很快，但在准确预测物理现象方面仍然有很多不足。这项研究为未来在物理模拟中使用智能算法提供了基础和方向。
2025-03-07｜TencentPCG-ARCLab,CUHK｜⭐️⭐️
http ://arxiv.org/abs/2503.05638v1https ://trajectorycrafter.github.io
TrajectoryCrafter是一种创新的方法，旨在为单目视频重新导向相机轨迹，以实现用户定义的视图变换和一致的4D内容生成。该方法通过解耦确定性视图变换与随机内容生成，提供了对用户指定相机轨迹的精确控制。传统的视频重建技术在处理遮挡区域时往往产生不切实际的结果，而TrajectoryCrafter通过结合点云渲染和源视频的双流条件视频扩散模型，克服了这一挑战。该模型不仅保证了视图变换的准确性，还确保了生成内容在4D空间中的一致性。通过构建一个混合训练数据集，结合了动态的网络规模单目视频和静态多视角数据集，TrajectoryCrafter展示了在多样场景中的强大泛化能力。
TrajectoryCrafter的核心方法包括几个关键步骤。首先，输入的视频被提升为动态点云，通过单目深度估计生成深度图。接着，利用逆透视投影将源视频与深度信息结合，构建出动态点云数据。然后，基于用户指定的相机轨迹，渲染出新的视图。为了实现精确的相机轨迹控制，TrajectoryCrafter采用了双流条件视频扩散模型，将点云渲染和源视频作为条件输入。该模型通过处理视图令牌和参考令牌，利用交叉注意力机制增强生成视频的细节和一致性。此外，模型的训练分为两个阶段，第一阶段侧重于合成缺失区域，第二阶段则通过引入静态多视角数据提高生成视频的质量和一致性。
在实验部分，TrajectoryCrafter在多个数据集上进行了广泛评估，包括多视角视频基准和大规模单目视频数据集。通过使用PSNR、SSIM和LPIPS等指标，量化生成视频的质量。实验结果表明，TrajectoryCrafter在生成高保真度视频和精确控制相机轨迹方面表现优异，显著优于现有的基线方法。此外，模型在处理复杂场景时，能够有效保持4D一致性，展现出良好的泛化能力。通过与其他生成模型的比较，TrajectoryCrafter在视觉质量、运动一致性和背景一致性等方面均取得了显著提升，验证了其在实际应用中的潜力和优势。
TrajectoryCrafter的工作原理可以简单理解为几个步骤。首先，它从一段视频中提取出深度信息，形成一个动态的三维点云。接着，用户可以指定想要的相机移动路径，比如放大或旋转。系统会根据这个路径生成新的视图，让视频看起来像是从不同的角度拍摄的。同时，TrajectoryCrafter结合了源视频的细节和点云的结构，确保生成的新视频不仅清晰，而且与原视频保持一致。为了提高效果，系统分两步进行训练，第一步让模型学习如何填补视频中的空白，第二步则增强生成视频的质量。这样的设计使得TrajectoryCrafter能够在各种场景中生成高质量的视频，给用户带来更好的观看体验。
2025-03-07｜CUHK,TencentPCG-ARCLab,UTokyo,UMacau｜⭐️🟡
http ://arxiv.org/abs/2503.05639v1https ://yxbian23.github.io/project/video-painter/
VideoPainter是一种新颖的双分支视频修复和编辑框架，旨在处理任意长度的视频内容。该方法通过引入轻量级上下文编码器，显著提升了对背景的保持和前景生成的能力。传统的视频修复方法往往面临在长视频中保持对象一致性和处理复杂遮挡的挑战，而VideoPainter通过分离背景保持与前景生成的过程，有效解决了这些问题。此外，研究团队还构建了VPData和VPBench，这是迄今为止最大的带有分割掩码和详细视频说明的数据集，为大规模训练和评估提供了支持。通过广泛的实验，VideoPainter在视频质量、掩码区域保持和文本一致性等多个关键指标上均表现出色，展示了其在视频编辑和修复领域的潜力。
VideoPainter的核心方法包括三个主要部分：1）VPData和VPBench构建流程，2）双分支视频修复控制，3）目标区域ID重采样。首先，为了应对现有数据集规模不足和缺乏文本注释的问题，研究团队利用先进的视觉模型构建了VPData和VPBench，经过五个步骤的处理，包括数据收集、注释、分割、选择和标注，最终形成了超过390K的视频片段。其次，VideoPainter利用上下文编码器提取被遮盖视频特征，并将其与预训练的扩散变换器结合，实现背景与前景的分离生成。最后，ID重采样技术确保了在长视频中的对象一致性，通过在训练和推理过程中重采样掩码区域的ID信息来维护视频的连贯性和一致性。
在实验部分，VideoPainter的性能通过与现有最先进的基线进行对比评估。研究团队在VPBench和DAVIS数据集上进行了定量比较，使用了多种指标来评估修复质量，包括PSNR、SSIM、LPIPS、MSE和MAE等。结果显示，VideoPainter在掩码区域保持、文本一致性和视频生成质量等方面均超过了传统方法，如ProPainter和COCOCO等。此外，用户研究表明，参与者对VideoPainter在背景保持、文本对齐和视频质量方面的偏好显著高于其他基线。通过消融实验，研究团队进一步验证了双分支架构和ID重采样在提升性能方面的重要性，证明了VideoPainter在视频修复和编辑任务中的有效性。
VideoPainter的工作原理可以简单理解为一个视频编辑工具，它能够让用户在视频中添加、删除或更改对象。首先，它会收集大量的视频数据，并为每个视频打上标签，告诉系统视频中有哪些内容。接下来，VideoPainter使用一种特殊的技术，能够同时关注视频的背景和前景。这样，即使在复杂的场景中，它也能确保背景不被破坏，同时生成新的前景元素。最后，在处理长视频时，它会保持对象的一致性，确保视频中的人物或物体在不同的场景中看起来是相同的。总的来说，VideoPainter就像一个智能的视频修复助手，帮助用户轻松实现视频的各种编辑需求。
2025-03-06｜Unknwon｜⭐️
http ://arxiv.org/abs/2503.04871v1https ://github.com/RedShift51/fast-latent-decoders
在本研究中，我们探讨了通过引入轻量级解码器来减少图像和视频生成中稳定扩散模型的推理时间和内存占用。传统的潜在扩散管道依赖于大型变分自编码器（VAE）解码器，导致生成速度缓慢和显著的GPU内存消耗。我们提出了自定义训练的解码器，采用轻量级的视觉变换器和驯化变换器架构。实验结果表明，在COCO2017数据集上，图像生成的整体速度提高了15%，而解码子模块的速度提升达到20倍，视频生成的性能也有所提升。尽管在感知质量上与默认解码器相比略有下降，但在大规模推理场景（如生成10万张图像）中，速度和可扩展性的提升至关重要。此外，我们的研究还与视频生成的效率提升相结合，展示了在生成模型领域改善可扩展性和效率的更广泛努力。
我们的方法主要集中在修改标准潜在扩散模型的解码器组件，同时保持UNet去噪过程不变。我们设计了两种轻量级解码器架构：
TAE-192：一种小型自编码器架构，使用diffusers和TamingTransformers库。
EfficientViT：一种压缩的视觉变换器解码器。此外，我们为视频模型训练开发了“TAE-192Temporal”变体，通过在空间层之间嵌入时序块，注入时序注意力层。我们的训练数据集包括LAION-Face、LAION-Aesthetics、COCO2017和UCF101，确保解码器在多样化数据上的有效性。训练策略采用复合重建损失，结合像素级保真度、感知相似性和对抗监督。通过这种方式，我们的解码器能够有效地将潜在表示反转为RGB帧，确保在保证质量的同时提高速度。
在实验部分，我们使用了五个关键评估指标来测量模型性能，包括结构相似性指数（SSIM）、峰值信噪比（PSNR）、FréchetInceptionDistance（FID）、平均解码时间和内存占用。针对COCO2017数据集，我们评估了三种图像解码器（基线SDXLVAE、TAE-192和EfficientViT），并在256×256和1024×1024分辨率下进行比较。结果显示，尽管TAE-192和EfficientViT的SSIM和PSNR低于基线，但它们的解码时间几乎减少了一半。在视频实验中，我们将标准的SVDVAE与我们的解码器进行比较，发现TAE-192的解码时间显著减少，尽管其FID较高。通过引入时序层，TAE-Temporal在速度和质量之间取得了良好的平衡，表明在视频生成中速度与感知质量之间存在权衡。
在我们的方法中，我们主要关注如何改进图像和视频生成模型的解码过程。我们没有改变整个模型的架构，而是专注于解码器部分。我们设计了两种新型解码器：一种是小型自编码器，另一种是高效的视觉变换器。这些解码器能够快速将模型生成的潜在表示转换为可视图像。为了确保我们的解码器能够在多种情况下有效工作，我们使用了多种数据集进行训练和测试。我们还采用了一种结合多种损失函数的训练策略，以确保生成图像的质量和速度。通过这些改进，我们的解码器在生成图像和视频时能够更快地工作，同时保持相对较高的质量，这对于需要快速生成大量内容的应用场景尤为重要。
